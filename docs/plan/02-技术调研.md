## 1. LangGraph + MCP 最新模式 

- **LangGraph** v1.0.6 稳定版，提供图优先编排：有状态节点、循环工作流、运行时图变异
- 架构模式：LangGraph Agent 通过适配器连接 MCP Server，自动发现工具并转为 LangChain 兼容格式
- 核心原语：**State**（TypedDict 共享数据）、**Nodes**（Agent 逻辑）、**Edges**（路由函数）
- 每步自动 checkpoint（MemorySaver / SqliteSaver / PostgresSaver），支持"时间旅行"调试

## 2. ReAct & Tool Calling 最佳实践 

- **ReAct**（Reason + Act）仍是主流 Agent 模式
- 现代工具调用使用**原生 function calling**（结构化 JSON 输出），取代早期 prompt hacking
- 生产要点：
  - 结构化输出只在最终步强制，中间步给模型灵活性
  - 完善测试套件（事实性 + 多步骤）
  - 监控 token 消耗和迭代次数
  - 边界情况错误处理

**参考**：

- [Building Production ReAct Agents From Scratch](https://www.decodingai.com/p/building-production-react-agents)
- [LangChain Tool-Calling Agent vs. ReAct Agent](https://medium.com/@dzianisv/)

## 5. 代码 RAG：向量 + 符号 + 图

**两大范式**：

1. **非图方法**：词法/语义相似度（dense embeddings）
2. **图方法**：AST/依赖图显式建模代码结构

**前沿项目**：


| 项目                     | 方法                                       | 效果                               |
| ---------------------- | ---------------------------------------- | -------------------------------- |
| **Code-Graph-RAG**     | Tree-sitter + Memgraph/Neo4j + UniXcoder | 意图搜索（"error handling functions"） |
| **CodeRAG (Bigraph)**  | 二部图建模代码实体 + 关系                           | Pass@1 +35.57                    |
| **cAST**               | AST-based chunking（非文本 chunking）         | 保持结构完整性                          |
| **Microsoft GraphRAG** | 向量搜索 + 知识图谱                              | 语义 + 结构                          |


**实践方案**：

- Tree-sitter 解析 AST → 提取实体（函数/类/模块）和关系（调用/导入/继承）
- 图 DB（Neo4j/Memgraph）存结构化关系
- 向量 DB（Qdrant/Chroma）存 embedding → 语义搜索
- 查询管线：意图 → 语义搜索候选 → 图遍历扩展上下文 → 重排 → 喂 LLM

## 6. 多 Agent 代码审查

**行业案例**：


| 产品                     | 架构                        | 效果                    |
| ---------------------- | ------------------------- | --------------------- |
| **Claude Code Review** | 多 Agent 并行分析 diff         | 54% PR 深度分析，40% 更快合并  |

**多 Agent 架构模式**：

1. Diff Analysis Agent — 理解改了什么
2. Security Agent — 扫描漏洞
3. Test Impact Agent — 识别受影响测试
4. Architecture Agent — 跨模块影响
5. **Judge Agent** — 聚合、去重、过滤
6. Remediation Agent — 生成修复补丁

## 7. 对本项目的启示

1. **工具要代码专属**：AST 搜索、编译执行、测试运行、Git 操作，不要天气/计算器
2. **用 LangGraph 不用手写**：JD 明确要框架经验，checkpoint/状态管理是加分项
3. **轨迹导出是差异化**：能说清 "我的 Agent 产出的数据可以用来 SFT/DPO" 就赢了
4. **多 Agent 是必须的**：Coder/Reviewer/Tester 分工，展示真实的 multi-agent 理解
5. **Tree-sitter 是性价比最高的 RAG 方案**：零延迟、确定性、不需要 embedding 模型
