# 03 - Coding Agent 工具层参考文档

> 阶段一产出：6 个代码专属 MCP 工具的设计与实现说明

---

## 概述

在已有 6 个基础工具（echo、calculate、get_time、get_weather、write_file、analyze_text_file）基础上，新增 6 个面向 Coding Agent 的工具，总计 12 个工具通过 MCP HTTP JSON-RPC 2.0 协议对外暴露。

所有新工具遵循统一的注册模式：

```
src/server/builtin/tools/{tool_name}_tool.cpp   ← 单文件实现
tool_registrars.h                                ← 函数声明
builtin_tools.cpp                                ← 调用注册
src/CMakeLists.txt                               ← 编译单元
```

---

## 工具一览

| # | 工具名 | 文件 | 用途 | 安全策略 |
|---|--------|------|------|---------|
| 1 | `read_file` | `read_file_tool.cpp` | 读取 workspace 内文件内容 | 沙箱 + 路径穿越防护 + 1MB 截断 |
| 2 | `list_files` | `list_files_tool.cpp` | 遍历目录，支持 glob 过滤 | 沙箱限制 + 1000 条上限 |
| 3 | `execute_code` | `execute_code_tool.cpp` | 在沙箱内执行 shell 命令 | 白名单 + 危险符号拦截 + 64KB 输出限制 |
| 4 | `run_tests` | `run_tests_tool.cpp` | 编译并运行测试文件 | 沙箱 + 结果解析 |
| 5 | `code_search` | `code_search_tool.cpp` | 代码关键词/符号搜索 | 只读，沙箱内搜索 |
| 6 | `git_diff` | `git_diff_tool.cpp` | Git 操作（diff/log/blame/status） | 只读 git 命令 |

## 1. read_file

**功能**：读取 workspace/ 沙箱内的文件内容。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `path` | string | 是 | 相对于 workspace/ 的文件路径 |

**返回**：

```json
{
  "content": "文件内容...",
  "path": "/abs/path/workspace/file.py",
  "size": 1234,
  "truncated": false
}
```

**安全策略**：

- `std::filesystem::weakly_canonical()` 解析路径后，检查是否以 workspace/ 绝对路径为前缀
- `../../etc/passwd` 等路径穿越 → 返回 `"Error: path is outside the workspace sandbox"`
- 文件超过 1MB → 只返回前 1MB，`truncated: true`
- 非普通文件（目录、符号链接）→ 拒绝

**调用示例**：

```bash
curl --noproxy localhost -X POST http://localhost:8089/jsonrpc \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/call",
       "params":{"name":"read_file","arguments":{"path":"project_notes.md"}},
       "id":1}'
```

## 2. list_files

**功能**：遍历 workspace/ 内目录，支持 glob 模式过滤和递归。

**参数**：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `directory` | string | 否 | `"."` | 相对于 workspace/ 的目录路径 |
| `pattern` | string | 否 | `"*"` | Glob 过滤模式（如 `*.py`、`*.cpp`） |
| `recursive` | bool | 否 | `false` | 是否递归子目录 |

**返回**：

```json
{
  "files": [
    {
      "path": "tasks/buggy.py",
      "is_directory": false,
      "size": 682,
      "modified_time": 1711018800
    }
  ],
  "count": 4,
  "directory": ".",
  "pattern": "*.py",
  "recursive": true
}
```

**实现细节**：

- Glob 转正则：`*` → `.*`，`?` → `.`，`.` → `\\.`
- 目录条目不受 pattern 过滤（始终显示）
- 结果上限 1000 条，超出后 `truncated: true`
- 文件修改时间通过 `stat()` 系统调用获取（兼容 macOS）

## 3. execute_code

**功能**：在 workspace/ 目录下执行白名单内的 shell 命令。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `command` | string | 是 | 要执行的命令（必须使用白名单命令） |

**返回**：

```json
{
  "output": "hello\n",
  "exit_code": 0,
  "truncated": false
}
```

**安全策略详解**：

| 策略 | 实现 | 说明 |
|------|------|------|
| 命令白名单 | 前缀匹配 + 边界检查 | 见下方白名单列表 |
| 危险符号拦截 | 字符串扫描 | `;` `&&` `\|\|` `\|` `>` `<` `>>` `$(` `${` `` ` `` |
| 输出截断 | 读取时限制 | 最大 64KB |
| 工作目录 | `cd workspace/` | 所有命令在沙箱内执行 |

**白名单命令**：

```
python3  python  node
g++  gcc  clang++  clang
ls  cat  grep  rg
head  tail  wc  find
echo  pwd  stat  file
sort  uniq  diff  tr
```

**拦截示例**：

```
"rm -rf /"       → Error: command is not in the whitelist
"ls; cat /etc/p" → Error: command contains forbidden characters
"echo $(whoami)" → Error: command contains forbidden characters
```

## 4. run_tests

**功能**：编译并运行测试文件，解析测试结果（通过/失败数）。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `test_file` | string | 是 | 相对于 workspace/ 的测试文件路径 |
| `language` | string (enum) | 是 | `"python"` 或 `"cpp"` |

**返回**：

```json
{
  "output": "... pytest 输出 ...",
  "exit_code": 1,
  "language": "python",
  "test_file": "tasks/test_buggy.py",
  "test_results": {
    "passed": 3,
    "failed": 3,
    "errors": 0,
    "total": 6
  },
  "all_passed": false,
  "truncated": false
}
```

**执行策略**：

| 语言 | 执行方式 |
|------|---------|
| Python | `python3 -m pytest {file} -v --tb=short` |
| C++ | `g++ -std=c++17 {file} -o /tmp/mcp_test_xxx && /tmp/mcp_test_xxx` |

**结果解析**：

- Python：正则匹配 pytest 输出中的 `X passed`、`X failed`、`X error`
- C++：匹配 GTest 格式 `[  PASSED  ] X tests`；无法匹配时按 exit_code 判断（0 = 1 passed，非 0 = 1 failed）

## 5. code_search

**功能**：在 workspace/ 内搜索代码，支持两种模式。

**参数**：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `query` | string | 是 | — | 搜索关键词或符号名 |
| `mode` | string (enum) | 否 | `"keyword"` | `"keyword"` 或 `"symbol"` |
| `file_pattern` | string | 否 | `"*"` | Glob 过滤文件（如 `*.py`） |
| `top_k` | int | 否 | `10` | 最大返回条数 |

**keyword 模式返回**：

```json
{
  "results": [
    {
      "file": "tasks/buggy.py",
      "line": 4,
      "snippet": "def sort_list(lst):",
      "context": "3: \n4: def sort_list(lst):\n5:     ...\n6:     ...\n7:     ..."
    }
  ],
  "total_matches": 5,
  "mode": "keyword",
  "query": "sort_list"
}
```

**symbol 模式返回**（额外包含 `type` 和 `name`）：

```json
{
  "results": [
    {
      "file": "tasks/buggy.py",
      "line": 4,
      "type": "function",
      "name": "sort_list",
      "snippet": "def sort_list(lst):",
      "context": "..."
    }
  ]
}
```

**符号提取规则**（轻量级 regex，不依赖 Tree-sitter）：

| 语言 | 提取的符号类型 |
|------|-------------|
| `.py` | `def func_name(` → function，`class ClassName` → class |
| `.cpp/.h/.c/.cc/.hpp` | `返回类型 func_name(` → function，`class/struct Name` → class/struct |
| `.js/.ts/.jsx/.tsx` | `function name(`、`class Name`、`const name = () =>` → function/class |

**上下文**：每个匹配结果附带目标行前后各 3 行代码。

## 6. git_diff

**功能**：在项目仓库上执行只读 Git 操作。

**参数**：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `operation` | string (enum) | 是 | — | `"diff"` / `"log"` / `"blame"` / `"status"` |
| `path` | string | 否 | `""` | 文件路径（diff/blame 时可指定） |
| `n` | int | 否 | `5` | log 模式显示条数 |

**各操作返回**：

| 操作 | 返回字段 |
|------|---------|
| `diff` | `{operation, staged, unstaged, path?}` — 分别返回已暂存和未暂存的 diff |
| `log` | `{operation, output, count}` — `git log --oneline -n N` |
| `blame` | `{operation, path, output}` — 需要 path 参数 |
| `status` | `{operation, output}` — `git status --short` |

**输出限制**：所有 git 命令输出上限 64KB。

## 安全策略汇总

```
                        ┌───────────────────────────────────┐
                        │       workspace/ 沙箱边界          │
                        │                                   │
  read_file ──────────►│  canonical() 路径验证               │
  list_files ─────────►│  前缀匹配 sandbox_root              │
  write_file ─────────►│  拒绝 ../ 穿越                     │
  execute_code ───────►│  cd workspace/ && 命令             │
  run_tests ──────────►│  sandbox_root / test_file          │
  code_search ────────►│  recursive_directory_iterator      │
                        │                                   │
                        └───────────────────────────────────┘

  execute_code 额外防线:
  ┌─────────────────────────────────────────────────────┐
  │  用户输入 command                                     │
  │    ↓                                                 │
  │  [1] 危险符号扫描 (;  &&  ||  |  >  <  $()  `)       │
  │    ↓ 通过                                            │
  │  [2] 白名单前缀匹配 (python3, ls, grep, g++ ...)     │
  │    ↓ 通过                                            │
  │  [3] 在 workspace/ 下执行，输出 ≤ 64KB               │
  └─────────────────────────────────────────────────────┘
```
