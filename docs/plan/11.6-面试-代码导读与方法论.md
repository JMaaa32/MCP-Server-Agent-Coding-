## 三、AI 协作方法论（面试话术）

### 怎么回答"你怎么用 AI 的？"

> 我的工作方式是"架构师 + AI 编码"模式：
>
> 1. **我做设计决策**：分析 JD → 确定架构 → 定义模块接口 → 选技术栈
> 2. **AI 做实现**：根据我的 spec 生成代码，我做 code review
> 3. **我做 Debug**：AI 生成的代码在实际环境中会遇到各种问题（代理、平台差异、API 格式不兼容），我负责定位根因和设计修复方案
> 4. **迭代优化**：跑 Demo → 分析轨迹数据 → 发现瓶颈 → 改进
>
> 这其实就是 AI Coding 岗位的核心技能——不是替代编程，而是高效驾驭 AI 编程工具，同时保持对代码的深度理解和控制力。

### 三个具体例子说明"我不是让 AI 全部代写"

**例子 1：AI 搞不定的环境问题**

> Ollama 502 错误，AI 尝试了 3 种修复方案都失败。是我用 curl 排查出代理劫持 localhost，决定换库。AI 不了解我的运行环境。

**例子 2：AI 遗漏的安全漏洞**

> AI 实现 read_file 时加了沙箱验证，但实现 write_file 时忘了。是我 review 时发现路径穿越漏洞。AI 不会主动做跨文件的一致性检查。

**例子 3：AI 的 Prompt 导致效率问题**

> AI 写的 System Prompt 鼓励 Agent 先探索项目结构，导致每次任务多花 10 步。是我分析轨迹数据发现这个模式，重新设计了 Prompt 策略。AI 不会自我分析自己生成的 Prompt 的效果。
