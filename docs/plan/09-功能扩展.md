## 一、analyze_code 工具（C++ 层）

### 新增文件

`src/server/builtin/tools/analyze_code_tool.cpp`

### 功能

分析源代码文件的结构和复杂度指标：

| 指标 | 说明 |
|------|------|
| total_lines | 总行数 |
| code_lines | 非空非注释行 |
| comment_lines | 注释行 |
| blank_lines | 空行 |
| comment_ratio | 注释率 (comment_lines / code_lines) |
| function_count | 函数定义数 |
| class_count | 类/结构体定义数 |
| max_nesting_depth | 最大嵌套深度（Python 用缩进，C++ 用大括号） |
| cyclomatic_complexity | 圈复杂度估算（分支语句计数 + 1） |
| complexity_rating | low / moderate / high / very_high |

### 支持语言

- Python (.py)：检测 `def`/`class`/`import`，按缩进估算嵌套
- C/C++ (.cpp/.h/.c)：检测函数/class/struct/`#include`，按大括号深度估算嵌套

### 圈复杂度计算

```
cyclomatic_complexity = 1 + count(if/elif/for/while/except/and/or)  // Python
cyclomatic_complexity = 1 + count(if/else if/for/while/switch/case/catch)  // C++
```

评级:
- low: ≤ 5
- moderate: 6-15
- high: 16-30
- very_high: > 30

### 调用示例

```json
{
  "tool": "analyze_code",
  "arguments": {"path": "tasks/buggy.py"}
}
```

返回：

```json
{
  "file": "tasks/buggy.py",
  "language": "python",
  "metrics": {
    "total_lines": 27,
    "code_lines": 18,
    "cyclomatic_complexity": 5,
    "complexity_rating": "low",
    "max_nesting_depth": 2,
    "function_count": 2
  },
  "functions": ["sort_list", "find_max"],
  "imports": []
}
```

## 二、RAG 代码检索模块（Python 层）

### 新增文件

`agent/rag.py`

### 架构

```
CodeRAG
├── index()          扫描 workspace/ → 切片 → 向量化
├── query(text)      查询 → 余弦相似度 → top-k 返回
├── save_index()     导出索引 JSON
└── get_stats()      返回索引统计

代码切片策略:
  Python:  按 def/class 定义边界切分
  C/C++:   按函数/类定义 + 大括号匹配切分
  其他:    固定窗口滑动切分 (30行, 步长20行)

向量化后端:
  TFIDFVectorizer (内置, 无需外部依赖)
    - 驼峰/下划线分词
    - TF-IDF 权重
    - 余弦相似度排序
```

### 关键类

| 类 | 说明 |
|-----|------|
| `CodeChunk` | 代码片段（文件路径、行范围、类型、内容、向量） |
| `TFIDFVectorizer` | 轻量级 TF-IDF（不依赖 sklearn） |
| `CodeRAG` | 主引擎：索引 + 检索 |

### 用法

```python
from rag import CodeRAG

rag = CodeRAG(workspace_path="../workspace")
n = rag.index()       # 建立索引
print(f"Indexed {n} chunks")

results = rag.query("排序算法", top_k=5)
for r in results:
    print(f"[{r['score']:.4f}] {r['file']}:{r['lines']} {r['name']}")
```

### CLI

```bash
cd agent
python rag.py ../workspace "sort"  # 索引 workspace，查询 "sort"
```

### 面试要点

> **JD 第 4 点**：结合 RAG，实现 Agentic 交互式结构化的代码检索
>
> 本模块实现了结构化代码检索：不是按行搜索（code_search 已实现），而是按**语义单元**（函数、类）切分代码，用 TF-IDF 向量化后通过余弦相似度排序，返回最相关的代码片段。
>
> 与 code_search 的区别：
> - code_search: 关键词/正则匹配，精确但需要知道确切名称
> - RAG: 语义检索，支持模糊查询（如 "排序算法" 可匹配 sort_list 函数）

## 三、SSE 实时推送模块

### 新增文件

`agent/sse.py`

### 功能

通过 Server-Sent Events 协议实时推送 Agent 执行状态，支持浏览器直接监控。

### 事件类型

| 事件 | 触发时机 | 数据字段 |
|------|---------|---------|
| `agent_start` | Agent 节点开始 | agent, iteration, task |
| `tool_call` | 工具调用发起 | agent, tool, args |
| `tool_result` | 工具返回结果 | agent, tool, is_error, preview |
| `agent_done` | Agent 节点完成 | agent, steps/verdict/passed |
| `iteration` | 一轮 C→R→T 完成 | iteration, passed |
| `task_complete` | 整个任务结束 | success, total_steps |

### 架构

```
SSEServer (后台线程, port 8765)
├── /events    SSE 事件流端点
├── /status    JSON 状态查询
└── /          HTML 监控页面（内置）

push() → 广播到所有连接的客户端
       → 保留最近 100 条事件历史
```

### 用法

```bash
# 启用 SSE 监控
python main.py --sse "创建 palindrome 函数..."

# 浏览器打开
open http://localhost:8765

# 或 curl 订阅
curl http://localhost:8765/events
```

### 集成位置

- `agent/main.py`: `--sse` 参数，启动 SSEServer
- `agent/nodes/coder.py`: 推送 agent_start/tool_call/tool_result/agent_done
- `agent/nodes/reviewer.py`: 推送 agent_start/agent_done
- `agent/nodes/tester.py`: 推送 agent_start/agent_done

### 面试要点

> SSE 是单向的服务器推送协议，比 WebSocket 简单，适合实时日志/状态推送场景。浏览器端用 `new EventSource('/events')` 即可接收。实现上是在 Agent 主进程中跑一个后台线程的 HTTP 服务器，Agent 执行工具调用时通过全局单例 `sse_server.push()` 广播事件。
