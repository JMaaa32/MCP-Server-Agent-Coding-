# 01 - 基于自研 C++ MCP Server 的多 Agent Coding 系统 — 完整实施计划

> 目标 JD：AI算法工程师 - AICoding/Agents 方向

---

## 0. 项目定位

在已有 C++ MCP Server 框架基础上，构建一套 **Coding Agent 系统**：

- C++ 层：扩展 **13 个**代码专属 MCP 工具（代码搜索、沙箱执行、测试运行、Git 操作、复杂度分析）
- Python 层：基于 **LangGraph** 的多 Agent 编排（Coder / Reviewer / Tester）+ **DeepSeek API** 驱动
- 数据层：Agent 轨迹收集 → SFT / DPO 训练数据导出 + 效率评分 
- 扩展层：TF-IDF 代码 RAG 检索 + SSE 实时监控 

核心区别于通用 Agent：所有工具、Agent、Demo 任务都围绕 **代码** 场景设计。

## 5. 阶段一：C++ 工具层扩展 ✅ 

### read_file + list_files

- **read_file_tool.cpp**：`weakly_canonical()` + 前缀检查实现沙箱验证
- **list_files_tool.cpp**：`directory_iterator` + glob→regex 转换，`stat()` 替代 `file_time_type`（macOS 兼容）

### execute_code + run_tests

- **execute_code_tool.cpp**：
  - 命令白名单（`ALLOWED_PREFIXES`：python3, node, g++, gcc 等 18 个）
  - 引号感知状态机 `extract_unquoted_parts()`：4 状态（普通/单引号/双引号/转义），只检查引号外的危险符号
  - 危险符号检测 `contains_dangerous_patterns()`：拦截 `;`、`&&`、`||`、`|`、```、`$()`
- **run_tests_tool.cpp**：正则解析 pytest 输出，提取 passed/failed/total

### code_search + git_diff

- **code_search_tool.cpp**：
  - 关键词搜索：正则匹配 + `get_context()` 上下文提取
  - 符号搜索：`extract_symbols()` 正则匹配 def/class/function 定义
  - **注**：最初计划用 Tree-sitter AST，实际用正则实现（零依赖，精度够用）
- **git_diff_tool.cpp**：popen 执行 git diff/log/blame

### analyze_code（Phase 5 新增）

- **analyze_code_tool.cpp**：
  - `analyze_python()`：检测 def/class/import，缩进推算嵌套深度
  - `analyze_cpp()`：检测函数/类/includes，大括号追踪嵌套
  - 圈复杂度：CC = 1 + count(分支语句)，评级 low/moderate/high/very_high

## 6. 阶段二：Python LangGraph Agent 层 ✅

### llm.py — LLM 统一接口

- `chat()` 入口，backend 参数分发到 `_chat_deepseek()` / `_chat_ollama()`
- `make_tool_result_messages()`：处理 DeepSeek vs Ollama 消息格式差异
  - DeepSeek：需要 `tool_call_id`、`type: "function"`、`json.dumps(arguments)`
  - Ollama：简单 `{"role": "tool", "content": "..."}`
- 双层重试（DeepSeek）：连接错误自动重试

### graph.py — LangGraph 状态图

```
START → coder → reviewer → tester → should_continue
                                        ├─ "done"        → END
                                        ├─ "retry"       → coder
                                        └─ "max_reached" → END（3轮上限）
```

### nodes/ — 三个 Agent 节点


| 角色       | 工具上限 | 工具权限                                                                 | Prompt 特点      |
| -------- | ---- | -------------------------------------------------------------------- | -------------- |
| Coder    | 8 次  | read/write_file, code_search, list_files, execute_code, analyze_code | 3-5步完成 + 反模式约束 |
| Reviewer | 3 次  | read_file, code_search, list_files, analyze_code                     | 只关注正确性 bug     |
| Tester   | 5 次  | run_tests, read_file, execute_code                                   | 测试通过即停         |


### tool_adapter.py — Schema 适配 + 角色权限

- `mcp_tool_to_ollama()`：`inputSchema` → `parameters`，外层包 `{"type": "function", ...}`
- `filter_tools_for_role()`：按 coder/reviewer/tester 过滤可用工具

## 7. 阶段三：轨迹收集与训练数据 ✅

### trajectory.py — 轨迹收集

- `record_step()`：记录 thought/action/action_input/observation/is_error
- `export_sft_format()`：成功轨迹 → (instruction, output) 对，output 是完整 ReAct 链
- 质量分级：good（efficiency > 0.6）/ usable / rejected

### analyze.py — DPO + 效率评分

- `generate_dpo_pairs()`：
  - success_vs_failure：同任务成功 vs 失败
  - efficient_vs_inefficient：步数少 vs 步数多
- 效率评分公式：`efficiency = 1.0 - error_rate×0.3 - redundancy_rate×0.3 - step_penalty×0.4`

## 9. 阶段五：效率优化 + 功能扩展 ✅

### 效率优化（三个措施）

1. **差异化工具上限**：Coder 8 / Reviewer 3 / Tester 5（原来一刀切 15）
2. **Prompt 反模式约束**：禁止无目的探索、禁止重复调用、禁止读刚写的文件
3. **后续轮次精简反馈**：只给要修的点，不重复完整任务描述

### 功能扩展

1. **analyze_code**（C++ 工具）：圈复杂度、嵌套深度、注释率、函数/类列表
2. **RAG 代码检索**（Python）：TF-IDF 向量化 + 驼峰/下划线分词 + 余弦相似度，34 chunks 索引 9 文件
3. **SSE 实时推送**（Python）：后台 daemon 线程 HTTP 服务器，`/events` SSE 流 + `/status` JSON + `/` 监控页

## 10. 安全策略汇总


| 策略     | 适用工具                              | 实现方式                                  |
| ------ | --------------------------------- | ------------------------------------- |
| 沙箱路径限制 | read_file, write_file, list_files | `weakly_canonical()` + 前缀检查           |
| 路径穿越防护 | read_file, write_file             | 解析 `..` 和 symlink 后验证                 |
| 命令白名单  | execute_code, run_tests           | 18 个命令前缀匹配                            |
| 引号感知过滤 | execute_code                      | `extract_unquoted_parts()` 状态机，只检查引号外 |
| 危险符号拦截 | execute_code                      | 禁止 `;` `&&` `                         |
| 执行超时   | execute_code, run_tests           | `timeout 10s`                         |
| 输出截断   | execute_code, run_tests           | 64KB 上限                               |
| 文件大小限制 | read_file                         | 1MB 上限                                |
